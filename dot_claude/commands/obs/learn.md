# /obs:learn — Генерация и сохранение знаний

Автоматически генерирует инструкции, гайды, справки и статьи, сохраняя их в правильное место базы знаний Obsidian.

## Triggers (автоматический вызов)

Этот skill должен автоматически вызываться когда пользователь просит:
- Создать инструкцию / гайд / туториал / howto
- Написать справку / документацию
- Объяснить как что-то сделать
- Создать статью на техническую тему
- Сгенерировать чек-лист / шпаргалку

## Usage

```
/obs:learn <тема>
/obs:learn --type <тип> <тема>
/obs:learn --preview <тема>
```

**Аргументы:**
- `<тема>` — тема для генерации (обязательно)
- `--type <тип>` — явно указать тип (cheatsheet, wiki, snippet)
- `--preview` — показать содержимое без сохранения
- `--no-commit` — не делать git commit

## Behavioral Flow

### 1. Анализ запроса

Определить:
- **Тему** — о чём создать материал
- **Тип контента** — на основе ключевых слов и содержания
- **Целевую папку** — по PARA методологии

### 2. Определение типа контента

| Ключевые слова | Тип | Папка |
|----------------|-----|-------|
| инструкция, гайд, howto, туториал, как сделать | `cheatsheet` | `60 Resources/Cheatsheets/` |
| шпаргалка, команды, quick reference | `cheatsheet` | `60 Resources/Cheatsheets/` |
| концепция, что такое, теория, архитектура | `wiki` | `30 Wiki/` |
| код, скрипт, функция, пример кода | `snippet` | `60 Resources/Code Snippets/` |
| статья, обзор, сравнение | `wiki` | `30 Wiki/` |

### 3. Генерация контента

Создать качественный материал с:

**Для cheatsheet:**
- Краткое описание
- Пошаговые инструкции
- Примеры команд/кода
- Troubleshooting секция
- Связанные ссылки

**Для wiki:**
- Определение/введение
- Основные концепции
- Примеры использования
- Плюсы и минусы (если применимо)
- Ссылки на ресурсы

**Для snippet:**
- Описание назначения
- Код с комментариями
- Примеры использования
- Зависимости (если есть)

### 4. Формирование имени файла

Формат: `<Тема> - <Подтема>.md`

Примеры:
- `Git - Rebase vs Merge.md`
- `Docker - Multi-stage Builds.md`
- `Apple Shortcuts - Export Notes to Obsidian.md`

Правила:
- Использовать Title Case для английских слов
- Тире для разделения темы и подтемы
- Без специальных символов (кроме тире и пробелов)

### 5. Frontmatter

```yaml
---
created: YYYY-MM-DD
updated: YYYY-MM-DD
type: <тип>
tags: [<тип>, <тема>, <подтема>]
---
```

### 6. Валидация перед сохранением

Перед записью проверить:
- [ ] Файл с таким именем НЕ существует (иначе предложить другое имя)
- [ ] Контент не пустой (минимум 100 символов)
- [ ] Frontmatter корректен (type, created, tags)
- [ ] Нет broken links в [[wikilinks]]

Если `--preview` — показать содержимое и спросить подтверждение.

### 7. Сохранение и коммит

1. Записать файл в целевую папку
2. Показать пользователю путь и превью
3. Выполнить git commit:
   ```bash
   git add "<путь к файлу>"
   git commit -m "Add <тип>: <название>"
   git push
   ```
4. Вывести результат:
   ```
   ✅ Сохранено: 60 Resources/Cheatsheets/Topic Name.md
   ✅ Закоммитил и запушил
   ```

### 7. Обновление справочника (если нужно)

Если создан материал о Claude Code или skill:
- Обновить `30 Wiki/Claude Code Skills.md`

## Качество контента

**Требования к генерируемому контенту:**

1. **Практичность** — реальные примеры, рабочие команды
2. **Полнота** — все шаги, ничего не пропущено
3. **Структура** — логичное деление на секции
4. **Краткость** — без воды, только суть
5. **Актуальность** — современные подходы и инструменты

## Auto-invoke механизм

Этот skill помечен `auto-invoke: true`. Claude должен автоматически вызывать его когда:

1. Пользователь использует фразы из списка `triggers` в frontmatter
2. Запрос явно требует создания документации/инструкции

**Как работает:**
- Claude видит `auto-invoke: true` в frontmatter
- При совпадении паттерна запроса с triggers — применяет behavioral flow этого skill
- Пользователю НЕ нужно явно вызывать `/obs:learn`

**Примеры триггеров:**
- "создай инструкцию как..." → auto-invoke
- "напиши гайд по..." → auto-invoke
- "как настроить..." → НЕ auto-invoke (слишком общее)
- "объясни как сделать..." → auto-invoke

## Обработка ошибок

| Ситуация | Действие |
|----------|----------|
| Файл уже существует | Предложить: обновить, создать с суффиксом, отменить |
| Не удалось определить тип | Спросить у пользователя через AskUserQuestion |
| Git push failed | Показать ошибку, сохранить локально |
| Пустая тема | Запросить уточнение |
